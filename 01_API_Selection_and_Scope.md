# API Selection and Scope

## Introduction

For this assessment, I selected **JSONPlaceholder** as the target API. JSONPlaceholder is a free REST API created for testing and learning, making it a safe platform to perform an API security risk analysis. This assessment focuses on reviewing the API's security posture, identifying potential security risks, and recommending improvements while following ethical testing practices.

---

## API Information

- **API Name:** JSONPlaceholder
- **API Type:** REST API
- **Base URL:** https://jsonplaceholder.typicode.com

The API provides sample resources such as:

- Users
- Posts
- Comments
- Albums
- Photos
- Todos

---

## Assessment Scope

During this assessment, I focused on:

- Reviewing the API documentation.
- Testing public API endpoints using Postman.
- Inspecting requests and responses.
- Reviewing response headers.
- Assessing authentication and authorization.
- Identifying potential API security risks.
- Recommending security improvements.

The assessment was limited to safe, read-only testing. No attempts were made to exploit vulnerabilities, bypass security controls, or modify data.

---


## Methodology

The assessment was completed using the following process:

1. Selected a public API.
2. Reviewed the API documentation.
3. Tested API endpoints using Postman.
4. Inspected requests, responses, and headers.
5. Identified potential security risks.
6. Classified each risk based on its severity.
7. Documented the findings and recommended remediation steps.

---

## Conclusion

This phase defines the scope of the assessment and the approach that will be used throughout the project. The next phase focuses on testing the API endpoints and reviewing their security-related behavior.
